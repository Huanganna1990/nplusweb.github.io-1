gulp的搭建

前端自动化的工具有很多：
第一种Gulp+Webpack构建前端自动化
第二种：Yeoman, Bower, Grunt构建前端自动化
第三种百度和腾讯也开发过.
在这里我研究的是Gulp+Webpack，如有兴趣可以研究其他的。

gulp主要做了下面几个工作：
1.css转化合并压缩
2.图片的雪碧图合并和base64
3.文件md5计算与替换
4.热启动，浏览器自动刷新
webpack解决了：
1.模块化开发
2.静态文件处理两大问题

基本步骤

第一步：安装node.js(https://nodejs.org/en/)
说明：gulp是基于node.js，所以必须安装node.js
打开cmd
node -v查看安装的nodejs版本，出现版本号，说明刚刚已正确安装nodejs
npm -v查看npm的版本号

第二步：创建一个文件存放项目：gulpgroct
cd定位到目录，用法：cd + 路径

第三步：全局安装gulp
命令如下：npm install -g gulp
gulp -v 查看安装版本，如果有版本好证明成功了

第四步：新建package.json文件和gulpfile.js文件
说明：package.json是基于nodejs项目必不可少的配置文件，它是存放在项目根目录的普通json文件
gulpfile.js文件gulpfile.js是gulp项目的配置文件，是位于项目根目录的普通js文件（其实将gulpfile.js放入其他文件夹下亦可）

第四步：安装本地的gulp
命令如下：npm install —save-dev gulp
项目目录如下
说明：我们全局安装了gulp，项目也安装了gulp，全局安装gulp是为了执行gulp任务，本地安装gulp则是为了调用gulp插件的功能

第五步：安装需要的插件（gulp的插件http://www.gulpjs.com.cn/docs/recipes/）
命令如下：npm install gulp-less —save-dev
gulp-less ：表示的插件的名字

第五部：运行gulp
1、说明：命令提示符执行gulp 任务名称； 2、编译less：命令提示符执行gulp testLess； 3、当执行gulp default或gulp将会调用default任务里的所有任务(‘testLess’,’elseTask’)
